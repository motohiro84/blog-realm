# Claude Code 設定

## 実行確認のルール（必ず守ること）

ツールの実行確認を求める際は、以下を**全て日本語**で記述すること。

### 確認メッセージの形式

1. **何をするのか** を具体的に説明する
   - 対象ファイル・コンテナ・サービス名を明記する
   - 実行するコマンドや変更内容を平易な言葉で説明する

2. **危険性がある場合** は必ず明記する
   - データ削除・上書き・ロールバック不可の操作
   - コンテナ・サービスの停止を伴う操作
   - 本番環境に影響しうる操作
   - 例：「⚠️ 危険：このコマンドは〇〇を削除します。元に戻せません。」

3. **英語での確認は禁止**。ツール呼び出しの説明文・コメントは常に日本語にする。

### 注意事項

確認ダイアログのボタン（Allow / Deny など）は Claude Code のUI側が表示するものであり、
このファイルでは制御できない。制御できる範囲（説明文・descriptionフィールド）を必ず日本語にすること。

### 例

悪い例：
> Run docker compose up -d --build to rebuild the container.

良い例：
> `blog-public-client` コンテナを停止・削除し、Dockerfile からイメージを再ビルドして起動します。ビルドには数分かかります。
> ⚠️ 現在起動中のコンテナは一時停止されます。
